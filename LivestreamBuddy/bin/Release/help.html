FAQ


Why is LivestreamBuddy's interface so frakken ugly?

Because I'm lazy and it's very beta.


Why does it sometimes take a few moments before my viewer list populates?

The chat and viewer list utilize twitch's IRC servers. Sometimes, when they are under high load, they will take a few moments to return data.


Why do I see a browser popup sometimes when you already have my credentials?

Short answer: Blame twitch.

Long answer: LivestreamBuddy actually uses two separate twitch interfaces. The chat and viewer list come over twitch's IRC servers and everything else comes from their web API. These two systems do not utilize the same authentication interface.


What should I do if I find a bug or have a feature request?

Go here and use the "Tracking" feature, there should be a button or tab or something... you probably need to be logged into SourceForge.


I entered a bug or feature request using SourceForge's "Tracking" feature like you told me to. Why isn't it done yet?

I'm lazy. You want it done faster? There's a donate button. Money makes me go faster.


Why is LivestreamBuddy all of a sudden using so much memory and CPU?

The chat now displays emoticon images. This has the potential to be resource intensive on slower machines. You can disable it by running LivestreamBuddy with the -noemote flag.


I see some chat text that should be emoticons. How do I make that text show up as emoticons?

LivestreamBuddy stores a list of emoticons from Twitch. Twitch may update that list on their servers periodically. You can force LivestreamBuddy to obtain a new list from Twitch by running with the -refreshemoticons flag.
